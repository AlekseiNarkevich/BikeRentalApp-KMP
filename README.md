# Bike Rental App (Flutter) — Lab 9

![Build Status](https://github.com/AlekseiNarkevich/BikeRentalApp-KMP/actions/workflows/main.yml/badge.svg)

Кроссплатформенное приложение **«Велопрокат»**, разработанное в рамках **Лабораторной работы №9 (Задание 4)** на базе **Flutter 3.24+ / Dart 3.5+**.  
Цель — модификация проекта из Лаб 8 в единую кодовую базу, работающую на Android, iOS, Linux (Desktop) и Web с адаптивным дизайном и полной бизнес-логикой.

Вариант задания: **9** (Поиск станций, бронирование нескольких велосипедов, оплата и отслеживание времени).

---

## 🚀 Функциональные возможности
- **Авторизация и сессии:**
  - Строгая логика: первый вход в систему считается регистрацией, последующие — проверкой пароля.
  - Сохранение сессии пользователя в **Hive** (автоматический вход при запуске отключен по ТЗ).
- **Главный экран (Stations):**
  - Представлен в виде адаптивной сетки карточек данных.
  - Оптимистичное обновление (Optimistic UI) для мгновенной реакции на бронирование в Web.
- **Бронирование (Booking):**
  - Возможность выбора количества велосипедов (от 1 до максимума на станции) с правильным склонением (1 велосипед, 2 велосипеда, 5 велосипедов).
  - Уведомления (Toast) о бронировании на всех платформах.
- **Интерактивная карта:**
  - Отображение всех 8 станций на карте (OpenStreetMap).
  - Визуальная индикация доступности (красные/серые маркеры).
- **Погода (API):**
  - На детальном экране станции отображается актуальная температура и иконка погоды (OpenWeatherMap API).
- **История аренды:**
  - Список активных бронирований с таймером реального времени.
  - Возможность завершения аренды через кнопку или жест смахивания (Swipe to dismiss).
- **Локализация:** **Русский / English / Беларуская**, динамическое переключение в настройках.
- **Темы:** Темная, светлая и системная темы с сохранением выбора.

## 🛠 Технологии
- **State Management:** `flutter_riverpod` (Riverpod 2.x)
- **Navigation:** `go_router` (поддержка URL в Web, Guard-редиректы)
- **Networking:** `dio` (запросы к API погоды и станций)
- **Local Storage:** `hive` + `hive_flutter` (кроссплатформенное NoSQL хранилище)
- **Animations:** `animations` (OpenContainer переход "развертывание")
- **Adaptive UI:** `flutter_adaptive_scaffold`, `LayoutBuilder`, `Context Extensions`

## 📱 Адаптация под платформы
- **Android / iOS:**
  - Нижняя навигация `BottomNavigationBar`.
  - Жесты пролистывания (свайп) для удаления бронирований.
- **Linux / Desktop (Windows):**
  - Боковое меню `NavigationRail`.
  - Горячие клавиши: `Ctrl + Q` для выхода из аккаунта (Shortcuts).
  - Поддержка мыши: открытие деталей правой кнопкой мыши (`onSecondaryTap`).
- **Web:**
  - Адаптация сетки под ширину окна (1, 2 или 3 колонки).
  - Корректная работа с URL маршрутами и историей браузера.
  - Постоянные скроллбары для удобства навигации мышью.

## 🧪 Тестирование (3–7 тестов каждого типа)
Проект содержит полное тестовое покрытие:
- **Unit tests** (`test/unit_test.dart`) — 7 тестов (бизнес-логика провайдеров и моделей).
- **Widget tests** (`test/widget_test.dart`) — 5 тестов (проверка UI-компонентов и настроек).
- **Integration tests** (`integration_test/app_test.dart`) — 4 сценария (E2E: регистрация -> вход -> навигация).

### Запуск тестов
Unit и Widget:
```bash
flutter test
```

Integration (нужен запущенный эмулятор):
```bash
flutter test integration_test/app_test.dart
```
